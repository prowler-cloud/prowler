## Overview

The Prowler App includes a **Category** filter in the Findings page that allows you to filter security findings by their assigned categories such as `internet-exposed`, `encryption`, `logging`, and more.

## Using the Category Filter

### Step 1: Navigate to Findings

1. Log in to Prowler App at http://localhost:3000
2. Click on **Findings** in the navigation menu

### Step 2: Apply Category Filter

1. Locate the **Category** filter in the filter controls section
2. Click on the Category dropdown
3. Select one or more categories (e.g., `internet-exposed`)
4. The findings table will automatically update to show only matching results

### Step 3: Combine with Other Filters

The Category filter works seamlessly with other filters:

- **Severity**: Filter by critical, high, medium, low, or informational
- **Status**: Filter by PASS, FAIL, or MANUAL
- **Provider**: Filter by AWS, Azure, GCP, Kubernetes, or M365
- **Region**: Filter by specific cloud regions
- **Service**: Filter by cloud services

## Example Use Cases

### View Internet-Exposed Resources

1. Select **Category**: `internet-exposed`
2. Select **Status**: `FAIL`
3. Select **Severity**: `critical`, `high`

This shows all critical and high severity failures for internet-exposed resources.

### Audit Encryption Compliance

1. Select **Category**: `encryption`
2. Select **Provider**: `aws`
3. Select **Status**: `FAIL`

This displays all AWS encryption-related failures.

## Available Categories

Common categories include:

- `internet-exposed` - Resources accessible from the internet
- `encryption` - Encryption configurations
- `logging` - Logging and monitoring
- `secrets` - Secrets management
- `forensics-ready` - Forensic readiness
- `trustboundaries` - Trust boundary violations

The available categories are dynamically populated based on your scan results.

## API Integration

The category filter uses the `categories__in` query parameter:

```
GET /api/v1/findings?categories__in=internet-exposed,encryption
```

This allows programmatic filtering via the Prowler API.
