# Create the Required Deployment Files

This page contains the local-only Cloud.gov deployment files that can be recreated when preparing a deployment or validating an upstream contribution.

Save these files in the listed locations before running `cf push`.

## Save as `cloudgov-vars.yml` at the Repository Root

Create this file locally and keep it out of version control.

```yaml
api_route: prowler-api-gsa-10x-prototyping.app.cloud.gov
uaa_client_id: replace-with-cf-service-key-client-id
uaa_client_secret: replace-with-cf-service-key-client-secret

uaa_login_redirect_url: /

django_secrets_encryption_key: replace-with-fernet-key
django_token_signing_key: |
  -----BEGIN PRIVATE KEY-----
  replace-with-rs256-private-key
  -----END PRIVATE KEY-----
django_token_verifying_key: |
  -----BEGIN PUBLIC KEY-----
  replace-with-rs256-public-key
  -----END PUBLIC KEY-----
```

```procfile

```

## Save as `api/runtime.txt`

```text
python-3.12.12
```

Use a Python version supported by the current Cloud Foundry Python buildpack. If staging rejects the version, update this file first.

## Save as `api/bin/cloudgov-web.sh`

```bash
#!/usr/bin/env bash
set -euo pipefail

cd "$(dirname "$0")/../src/backend"

export DJANGO_SETTINGS_MODULE="${DJANGO_SETTINGS_MODULE:-config.django.production}"
export DJANGO_BIND_ADDRESS="${DJANGO_BIND_ADDRESS:-0.0.0.0}"
export DJANGO_PORT="${PORT:-${DJANGO_PORT:-8080}}"
export NEO4J_HOST="${NEO4J_HOST:-}"
export NEO4J_PORT="${NEO4J_PORT:-}"
export NEO4J_USER="${NEO4J_USER:-}"
export NEO4J_PASSWORD="${NEO4J_PASSWORD:-}"

if [[ -z "${DATABASE_URL:-}" && -n "${VCAP_SERVICES:-}" ]]; then
  export DATABASE_URL="$(python - <<'PY'
import json
import os

services = json.loads(os.environ["VCAP_SERVICES"])
credentials = (services.get("aws-rds") or [{}])[0].get("credentials") or {}
print(credentials.get("uri", ""), end="")
PY
)"
fi

if [[ "${RUN_DB_MIGRATIONS:-0}" == "1" ]]; then
  python manage.py migrate
fi

gunicorn -c config/guniconf.py config.wsgi:application
```

export NEO4J_USER="${NEO4J_USER:-}"

  export DATABASE_URL="$(python - <<'PY'
import json
import os

## Mark the Shell Scripts as Executable

After saving the files, run:

```console
chmod +x api/bin/cloudgov-web.sh api/bin/cloudgov-worker.sh api/bin/cloudgov-beat.sh
```

## Deploy with the Local Files

After creating the files, deploy with:

```console
cf push --vars-file cloudgov-vars.yml
```

For incremental debugging, start with:

```console
cf push prowler-api-web --vars-file cloudgov-vars.yml
```

Run migrations explicitly as a task when needed:

```console
cf run-task prowler-api-web -k 4G -m 1G \
  --command "sh -lc 'cd src/backend && RUN_DB_MIGRATIONS=1 python manage.py migrate'"
```
